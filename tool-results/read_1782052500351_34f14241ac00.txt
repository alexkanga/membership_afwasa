   791→MESURES ET CALCULS À IMPLÉMENTER
   792→
   793→Créer un service backend ou une couche SQL qui calcule les KPI.
   794→
   795→Les calculs doivent respecter ces règles :
   796→
   797→-   membres payés = lignes où statut_paiement = Payé ;
   798→-   membres non payés = lignes où statut_paiement ≠ Payé ;
   799→-   membres actifs = lignes où statut_activation = Actif ;
   800→-   membres uniques = distinct email_normalise ;
   801→-   montant payé = somme montant des membres payés ;
   802→-   montant à recouvrer = somme montant des membres non payés ;
   803→-   taux paiement = membres payés / total inscriptions ;
   804→-   taux recouvrement = montant payé / montant attendu ;
   805→-   taux activation = membres actifs / total inscriptions ;
   806→-   taux complétude email = lignes avec email / total lignes ;
   807→-   taux complétude code membre = lignes avec code membre / total lignes ;
   808→-   pays représentés = distinct pays_normalise ;
   809→-   membres Afrique = zone_geographique = Afrique ;
   810→-   membres hors Afrique = zone_geographique = Hors Afrique.
