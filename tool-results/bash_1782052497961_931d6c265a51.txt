TU ES UN AGENT IA FULLSTACK SENIOR, EXPERT EN DÉVELOPPEMENT WEB, DATA DASHBOARD, BUSINESS INTELLIGENCE, AUTHENTIFICATION, BASE DE DONNÉES, IMPORT EXCEL, API BACKEND, FRONTEND MODERNE, UI/UX ET DÉPLOIEMENT.

Tu dois développer une application web complète permettant à l’Association Africaine de l’Eau et de l’Assainissement, AAEA / AfWASA, de piloter ses membres à partir d’un fichier Excel template régulièrement uploadé par l’administration.

L’application doit fonctionner comme un tableau de bord complet des membres, cotisations, paiements, recouvrements, catégories, pays, qualité des données, risques et historique des fichiers importés.

CONTEXTE MÉTIER

L’AAEA dispose d’un fichier Excel template nommé généralement :

20260617 LISTE DES INSCRIPTIONS ET POTENTIELS_TEMPLATE.xlsx

La feuille principale à exploiter est :

INSCRIPTIONS ET ADHESIONS

Les colonnes attendues dans cette feuille sont :

-   PLAN_ADHESION
-   SOCIETE / INSTITUTION / UNIVERSITE
-   PAYS
-   EMAIL
-   MONTANT
-   DATE INSCRIPTION
-   NUM FACTURE
-   MODE PAIEMENT
-   DATE PAIEMENT / ACTIVATION
-   ID
-   EST ACTIVE ?
-   TYPE ADHESION
-   CODE MEMBRE
-   DATE PAIEMENT COMPTABILITE
-   A PAYE ?

L’application doit permettre de charger ce fichier Excel à chaque mise à jour de la base membres. Après chaque upload, le dashboard doit être recalculé automatiquement.

OBJECTIF PRINCIPAL DE L’APPLICATION

Créer une application web sécurisée qui permet de :

1.  Se connecter avec login et mot de passe.
2.  Uploader le fichier Excel template.
3.  Valider la structure du fichier.
4.  Lire automatiquement les données de la feuille INSCRIPTIONS ET ADHESIONS.
5.  Nettoyer et normaliser les données.
6.  Calculer tous les KPI membership.
7.  Afficher un dashboard complet et professionnel.
8.  Suivre l’historique des fichiers uploadés.
9.  Gérer les erreurs de fichier et les anomalies.
10. Exporter ou consulter les statistiques.
11. Séparer l’espace administration de l’espace consultation.
12. Donner au Directeur Exécutif une vue claire, synthétique et décisionnelle.

RÈGLES DE NETTOYAGE DES DONNÉES

Lors de l’import, appliquer les règles suivantes.

1.  EMAIL

-   supprimer les espaces ;
-   convertir en minuscules ;
-   vérifier le format email ;
-   créer email_normalise ;
-   détecter les doublons sur email_normalise.

2.  PAYS

-   supprimer espaces inutiles ;
-   harmoniser les noms ;
-   créer pays_normalise ;
-   créer zone_geographique : Afrique / Hors Afrique ;
-   créer region_afrique si possible.

3.  PLAN_ADHESION

-   supprimer espaces ;
-   harmoniser les libellés ;
-   créer plan_adhesion_normalise ;
-   créer categorie_membre : actif, affilié, individuel, honneur, autre ;
-   créer sous_categorie_membre si possible.

4.  MONTANT

-   convertir en nombre ;
-   remplacer les valeurs vides par 0 uniquement si cela est justifié ;
-   signaler les montants vides ou incohérents.

5.  A PAYE ?

-   si valeur = OUI, alors statut_paiement = Payé ;
-   sinon statut_paiement = Non payé / à vérifier.

6.  EST ACTIVE ?

-   si valeur = OUI, alors statut_activation = Actif ;
-   sinon statut_activation = Non actif / à vérifier.

7.  TYPE ADHESION

-   subscribe = Nouvelle adhésion ;
-   renew = Renouvellement ;
-   autre ou vide = À vérifier.

8.  MODE PAIEMENT

-   os_offline = Paiement offline ;
-   os_stripe = Paiement en ligne Stripe ;
-   autre = À vérifier.

9.  DATES

-   convertir les dates Excel en dates valides ;
-   contrôler les dates manquantes ;
-   signaler paiement antérieur à inscription ;
-   signaler comptabilisation antérieure au paiement ;
-   calculer les délais.

MESURES ET CALCULS À IMPLÉMENTER

Créer un service backend ou une couche SQL qui calcule les KPI.

Les calculs doivent respecter ces règles :

-   membres payés = lignes où statut_paiement = Payé ;
-   membres non payés = lignes où statut_paiement ≠ Payé ;
-   membres actifs = lignes où statut_activation = Actif ;
-   membres uniques = distinct email_normalise ;
-   montant payé = somme montant des membres payés ;
-   montant à recouvrer = somme montant des membres non payés ;
-   taux paiement = membres payés / total inscriptions ;
-   taux recouvrement = montant payé / montant attendu ;
-   taux activation = membres actifs / total inscriptions ;
-   taux complétude email = lignes avec email / total lignes ;
-   taux complétude code membre = lignes avec code membre / total lignes ;
-   pays représentés = distinct pays_normalise ;
-   membres Afrique = zone_geographique = Afrique ;
-   membres hors Afrique = zone_geographique = Hors Afrique.

RÈGLES DE MISE EN FORME CONDITIONNELLE

Appliquer les seuils suivants par défaut.

Taux paiement :

-   vert si >= 80 %
-   orange si entre 60 % et 79 %
-   rouge si < 60 %

Taux recouvrement :

-   vert si >= 85 %
-   orange si entre 65 % et 84 %
-   rouge si < 65 %

Taux activation :

-   vert si >= 80 %
-   orange si entre 60 % et 79 %
-   rouge si < 60 %

Doublons email :

-   vert si 0
-   orange si entre 1 et 10
-   rouge si > 10

Paiements non comptabilisés :

-   vert si 0
-   orange si entre 1 et 10
-   rouge si > 10

Créances anciennes de plus de 90 jours :

-   vert si < 5 %
-   orange si entre 5 % et 15 %
-   rouge si > 15 %
